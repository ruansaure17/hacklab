A deliberately vulnerable web application built with Flask for web security training.

HackLab was created to help students and aspiring security professionals practice identifying, analyzing, and documenting common web application vulnerabilities in a safe, controlled environment.


## Disclaimer

⚠️ This project is intended for educational purposes only.

⚠️ Do not deploy this application in production environments.

⚠️ Vulnerabilities are intentionally included to simulate real-world security issues.

## Features

* User authentication system
* User profiles
* Administrative panel
* Search functionality
* File upload system
* REST API endpoints
* Posts and comments

## Technologies

* Python
* Flask
* SQLite
* HTML
* CSS
* JavaScript

## Installation

Clone the repository:

```bash
git clone https://github.com/YOUR_USERNAME/hacklab.git
cd hacklab
```

Install dependencies:

```bash
pip install -r requirements.txt
```

Run the application:

```bash
python app.py
```

The application will be available at:

```text
http://localhost:5000
```

## Learning Objectives

The goal of this lab is to:

* Practice web application assessment
* Learn common attack vectors
* Improve Burp Suite skills
* Understand security misconfigurations
* Develop vulnerability reporting skills

No vulnerabilities are disclosed beforehand. The challenge is to discover and document them through testing and analysis.

## Recommended Tools

* Burp Suite Community Edition
* OWASP ZAP
* Firefox Developer Tools
* Kali Linux (optional)
