This project is a deliberately designed black-box environment for offensive security training.

Unlike guided labs, the vulnerabilities are intentionally unknown at the start of each exercise. The objective is to simulate real-world penetration testing scenarios where the attacker must:

- Discover attack surfaces
- Identify vulnerabilities
- Validate exploitability
- Develop and test payloads
- Document findings independently


## Disclaimer

⚠️ This project is intended for educational purposes only.

⚠️ Do not deploy this application in production environments.

⚠️ Vulnerabilities are intentionally included to simulate real-world security issues.

## Installation

Clone the repository:

```bash
git clone https://github.com/YOUR_USERNAME/hacklab.git
cd hacklab
```

Install dependencies:

```bash
pip install -r requirements.txt
```

Run the application:

```bash
python app.py
```

The application will be available at:

```text
http://localhost:5000
```

## Learning Objectives

The goal of this lab is to:

* Practice web application assessment
* Learn common attack vectors
* Improve Burp Suite skills
* Understand security misconfigurations
* Develop vulnerability reporting skills

No vulnerabilities are disclosed beforehand. The challenge is to discover and document them through testing and analysis.

## Recommended Tools

* Burp Suite Community Edition
* OWASP ZAP
* Firefox Developer Tools
* Kali Linux (optional)
